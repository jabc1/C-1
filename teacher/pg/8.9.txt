回顾安装过程：
把光盘放入到光驱中，来到安装界面 
boot:回车
boot:linux text
   接下来会看到 加载光盘中的 vmlinuz initrd.img 到内存中运行 
    会运行一个叫anacronda的installer程序来引导我们一步一步选择进行系统的安装。
   这是交互式安装
 所谓的无人执守安装，指的是把要回答的问题先写到某一个文件中，让installer程序自己去读取答案。

－－－－－－－－－
192。168。1。254 服务器，负责共享无人执守安装配置文件。

  一堆 客户端主机需要到服务器上读取配置文件进行无人执守安装

服务器的配置有两步：
1 生成无人执守配置文件（kickstart程序来生成
[root@server ntfs]# yum -y install system-config-kickstart
[root@server ntfs]# system-config-kickstart 

生成的文件中，注意加上两段：
key --skip 

%packages
@core
@editors


2 把无人执守配置文件共享，让客户端可以读取到（我们来选择nfs方式共享 ）
]# ls -l /tmp/ks.cfg  权限是other是可r的
-rw-r--r-- 1 root root 1337 08-09 11:15 /tmp/ks.cfg

]# cat /etc/exports 
/tmp    *(ro)
]# /etc/init.d/nfs start


客户端主的配置是：
在boot:提示符下
boot:linux text xxxxxxxxxxx 指定无人执守文件的位置 


boot:linux ks=nfs:192.168.1.254:/tmp/ks.cfg

]# yum -y install *gnome* *xorg* *Xorg*


------系统启动过程-------------
目的：
1、系统是怎么启动的
2、系统出现 启动不起来的情况，你能知道启动到哪一步了，是谁出了问题。怎么解决?

过程：
1、开机，加电，自检，检查当前的所有硬件。
 控制权限在BIOS（basic input/output system）上。
2、读取第一启动顺序所设置的设备。比如硬盘。
 将硬盘启动，目的是为了找到boot loader程序，让它去加载操作系统启动必须的文件（内核）。读取硬盘的首扇区 （boot sector --- 512字节  不属于任何操作系统）


    446  MBR  主引导记录（这446字节的数据是你在fdisk给磁盘的时候分区时产生的，目的是找到活动分区）
    64   PT  分区表
    2   55 aa 磁盘是否为可用的标识

boot loader 安装时，可以装在两个地方：
在MBR上安装bootloader
在活动分区的首扇区上安装bootloader
